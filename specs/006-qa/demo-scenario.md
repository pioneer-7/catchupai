# 데모 시나리오

> **SSOT 문서** — 3분 데모 흐름, 말할 포인트, 시연 체크리스트

---

## 1. 3분 데모 흐름

| # | 동작 | 설명 | 시간 |
|---|------|------|------|
| 1 | 랜딩에서 CatchUp AI 소개 | 한 줄 정의 읽기 | 15초 |
| 2 | 샘플 데이터 로드 | `샘플 데이터로 시작` 클릭 | 10초 |
| 3 | 대시보드에서 위험 학생 수 확인 | KPI 카드 4개 설명 | 20초 |
| 4 | 학생 목록에서 가장 위험한 학생 클릭 | risk_score 내림차순 확인 | 15초 |
| 5 | 위험 원인 설명 확인 | AI 생성 요약 읽기 | 20초 |
| 6 | 회복학습 생성 클릭 | 버튼 클릭 + 생성 대기 | 15초 |
| 7 | 생성된 보강 플랜 확인 | 3단계 카드 설명 | 20초 |
| 8 | 개입 메시지 생성 및 복사 시연 | 생성 + 복사 버튼 | 15초 |
| 9 | 미니 진단 생성 및 1회 제출 | 3문항 응답 + 제출 | 30초 |
| 10 | 결과에 따라 위험 상태 업데이트 확인 | 점수 변화 + 배지 변화 | 20초 |

**총 예상 시간**: ~3분

---

## 2. 데모에서 강조할 포인트

- 단순 대시보드가 아니라 **개입 액션**까지 연결된다는 점
- AI가 설명과 회복 플랜을 만든다는 점
- 교육 현장의 반복 업무를 줄인다는 점
- 학생 상태가 진단 결과에 따라 **동적으로 변한다**는 점

---

## 3. 데모 전 체크리스트

- [ ] 배포 URL 정상 동작
- [ ] 샘플 데이터 로드 성공
- [ ] 대시보드 KPI 카드 정상 표시
- [ ] 학생 목록 → 상세 이동 정상
- [ ] 회복학습 생성 성공 (10초 이내)
- [ ] 메시지 생성 성공
- [ ] 미니 진단 생성 및 제출 성공
- [ ] 상태 업데이트 반영 확인

---

## 4. 데모 실패 시 대응

| 실패 지점 | 대응 |
|-----------|------|
| 샘플 데이터 로드 실패 | CSV 수동 업로드로 전환 |
| AI 생성 지연 | "실제로는 10초 이내" 언급 + fallback 결과 보여주기 |
| 상태 업데이트 미반영 | 새로고침 후 재확인 |
