# 엣지 케이스

> **SSOT 문서** — 입력 검증, AI 실패, 경계값, 예외 상황 정의

---

## 1. CSV 업로드 엣지 케이스

| 케이스 | 예상 동작 |
|--------|----------|
| 빈 CSV 파일 | 400 에러 + "데이터가 없습니다" |
| 필수 컬럼 누락 | 400 에러 + 누락 컬럼 목록 |
| 퍼센트 값 100 초과 | 400 에러 + 해당 행 표시 |
| 음수 값 | 400 에러 + validation error |
| 중복 학생 이름 | upsert (기존 데이터 갱신) |
| 매우 큰 CSV (100+행) | 정상 처리 (MVP에서는 소규모만 기대) |
| 한글 인코딩 깨짐 | UTF-8 기본, 에러 시 안내 |

## 2. 위험도 계산 엣지 케이스

| 케이스 | 예상 결과 |
|--------|----------|
| 모든 지표 양호 | risk_score = 0, `stable` |
| 모든 지표 최악 | risk_score = 100 (clamp) |
| 총합 100 초과 | 100으로 clamp |
| 경계값 (attendance_rate = 85) | +0 (≥85 조건) |
| 경계값 (missed_sessions = 2) | +25 |
| 복합 보정 조건 정확히 2개 | E1 적용 (+10) |
| 고위험 보정 조건 정확히 3개 | E2 적용 (+10) |

## 3. AI 생성 엣지 케이스

| 케이스 | 예상 동작 |
|--------|----------|
| AI API 타임아웃 | fallback 템플릿 표시 |
| AI 응답이 invalid JSON | fallback 템플릿 표시 |
| AI가 금지 표현 생성 | fallback 전환 또는 필터링 |
| AI가 4문항 생성 (3문항 요청) | 앞 3개만 사용 또는 재시도 |
| 강의자료 없이 회복학습 요청 | 일반 템플릿으로 생성 |
| risk_level = stable인 학생에 회복학습 요청 | 정상 처리 (제한 없음) |

---

## 4. 미니 진단 엣지 케이스

| 케이스 | 예상 동작 |
|--------|----------|
| 모든 문항 정답 (3/3) | risk_score -20 |
| 모든 문항 오답 (0/3) | risk_score 유지 |
| 빈 답안 제출 | validation error 또는 오답 처리 |
| 이미 제출된 진단에 재제출 | 거부 또는 새 진단 생성 안내 |
| risk_score가 이미 0일 때 감소 | 0으로 유지 (clamp) |
| risk_level이 stable인데 추가 감소 | 0 이하로 내려가지 않음 |

## 5. UI 엣지 케이스

| 케이스 | 예상 동작 |
|--------|----------|
| 학생 0명 대시보드 | "데이터 없음" + 업로드 유도 |
| 학생 목록 검색 결과 0건 | "검색 결과가 없습니다" |
| 존재하지 않는 학생 ID 접근 | 404 페이지 또는 목록으로 리다이렉트 |
| AI 생성 중 페이지 이탈 | 생성 취소 또는 백그라운드 완료 |
| 동시에 여러 AI 생성 요청 | 순차 처리 또는 마지막 요청만 반영 |
