# 사용자 흐름

> **SSOT 문서** — 운영자/교강사/학습자별 핵심 동선

---

## 1. 운영자 흐름

```
랜딩 → 샘플 데이터 로드 또는 CSV 업로드
  → 대시보드에서 위험 학생 수 확인
  → 학생 목록에서 위험 학생 정렬
  → 특정 학생 클릭 → 학생 상세
  → 위험 원인 확인
  → 개입 메시지 생성 → 복사 → 학생에게 안내
```

### 단계별 정리

1. 샘플 데이터 또는 CSV 업로드
2. 대시보드에서 위험 학생 확인
3. 특정 학생 클릭
4. 위험 원인 확인
5. 개입 메시지 생성
6. 학생에게 안내

---

## 2. 교강사 흐름

```
학생 목록 또는 대시보드
  → 특정 학생 상세 진입
  → 위험 원인 확인
  → 회복학습 킷 생성
  → 미니 진단 생성
  → 학생에게 제공
  → 결과 확인
```

### 단계별 정리

1. 특정 학생 상세 진입
2. 회복학습 킷 생성
3. 미니 진단 생성
4. 학생에게 제공
5. 결과 확인

---

## 3. 학습자 흐름 (데모용 간략화)

```
회복학습 내용 확인
  → 미니 진단 응시
  → 피드백 확인
  → 상태 개선 확인
```

### 단계별 정리

1. 회복학습 내용 확인
2. 미니 진단 응시
3. 피드백 확인
4. 상태 개선 확인

---

## 4. 데모 기준 핵심 동선 (CRITICAL)

> 이 동선이 끊기지 않는 것이 UX 최우선 기준이다.

```
랜딩 → 샘플 데이터 로드 → 대시보드 → 학생 목록 → 학생 상세
  → 회복학습 생성 → 메시지 생성 → 미니 진단 제출 → 상태 갱신
```
