# 디자인 원칙 및 카피 가이드라인

> **SSOT 문서** — UI 톤, 시각 원칙, 금지 표현, 핵심 시각 요소

---

## 1. 톤

- 차분함
- 교육적 신뢰감
- 과장 없는 AI 보조도구 느낌

---

## 2. UI 원칙

- 위험도는 **배지와 점수 모두** 표시
- 데이터 카드 중심 레이아웃
- 교강사/운영자가 빠르게 훑을 수 있어야 함
- **학생을 비난하는 표현 금지**

---

## 3. 핵심 시각 요소

| 요소 | 설명 |
|------|------|
| 상태 배지 | 안정 / 주의 / 위험 (색상 + 텍스트) |
| 우선순위 카드 | Top N 위험 학생 |
| 회복학습 단계 카드 | 3단계 복습 순서 |
| 복사 가능 메시지 블록 | 개입 메시지용 별도 UI |

---

## 4. 메시지 톤 가이드 (개입 메시지)

### 허용

- 격려형 톤
- 실행 유도
- 부담 완화형 표현

### 금지 (CRITICAL)

- 낙인 금지
- 비난 금지
- 협박/비교 표현 금지
- 공포심 유발 금지
- `반드시`, `심각하다`, `문제 학생` 같은 낙인형 표현 금지

### 좋은 예시

> 최근 학습 공백이 조금 생긴 것으로 보여서, 부담 없이 다시 시작할 수 있도록 짧은 따라잡기 플랜을 준비했어요.

---

## 5. 상태 라벨 정의

| risk_level | 한국어 라벨 | 색상 가이드 |
|-----------|------------|------------|
| `stable` | 안정 | 녹색 계열 |
| `warning` | 주의 | 노란/주황 계열 |
| `at_risk` | 위험 | 빨간 계열 |
