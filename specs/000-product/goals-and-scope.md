# MVP 목표 및 범위

> **SSOT 문서** — MVP 성공 조건, In/Out Scope, 개발 우선순위, 일정

---

## 1. MVP 목표

공모전 기간 내에 아래 한 가지 시나리오를 완성도 높게 보여준다.

> 운영자가 강의자료와 학습자 데이터를 업로드하면, CatchUp AI가 위험 학습자를 탐지하고, 특정 학생의 회복학습 킷과 개입 메시지를 생성하며, 미니 진단 결과에 따라 위험 상태를 업데이트한다.

## 2. MVP 성공 조건

다음이 가능해야 한다.

1. 샘플 데이터 업로드 가능
2. 위험 학습자 목록 확인 가능
3. 특정 학생 클릭 시 위험 원인 설명 확인 가능
4. 회복학습 킷 생성 가능
5. 개입 메시지 생성 가능
6. 미니 진단 생성 및 제출 가능
7. 제출 결과에 따라 상태 변경 가능

## 3. 범위 정의

### 3.1 In Scope

- 강의자료 업로드(PDF 또는 텍스트)
- 학습자 데이터 업로드(CSV)
- 위험도 계산
- 위험 원인 설명
- 회복학습 킷 생성
- 미니 진단 문제 생성
- 개입 메시지 생성
- 학생 상세 화면
- 간단한 대시보드

### 3.2 Out of Scope

- 실제 학교 시스템/LMS 연동
- 실시간 출결 자동 수집
- 학부모 알림 시스템
- 장기 성적 예측 모델
- 고도화된 추천 엔진
- 모바일 앱 네이티브 버전
- 다국어 지원
- 다중 기관 멀티테넌시

## 4. 개발 우선순위

### P0 (반드시)

- 업로드
- 대시보드
- 학생 목록
- 학생 상세
- 위험도 계산
- 회복학습 생성
- 개입 메시지 생성
- 미니 진단 및 결과 반영

### P1 (여유 시)

- 과정별 필터
- 샘플 데이터 템플릿 다운로드
- 회복학습 PDF 내보내기
- 학생별 히스토리 타임라인

### P2 (이번 대회 제외 권장)

- 실시간 연동
- 관리자 권한 체계
- 멀티 코스 지원 확장
- 정교한 ML 예측 모델

## 5. 7일 개발 일정

| Day | 주요 작업 |
|-----|----------|
| 1 | 문서 확정, 와이어프레임, 데이터 모델 확정, 기술 스택 세팅 |
| 2 | 업로드 기능, 샘플 데이터 시드, 대시보드 기본 UI |
| 3 | 학생 목록, 학생 상세, 위험도 계산 로직 |
| 4 | 강의자료 파싱, 위험 원인 설명 생성, 회복학습 생성 |
| 5 | 미니 진단 생성, 개입 메시지 생성, 결과 저장 |
| 6 | 상태 업데이트, UI 폴리싱, 에러 처리, README 작성 |
| 7 | 데모 리허설, 버그 수정, 제출 산출물 정리, AI 리포트 PDF |

## 6. 비기능 요구사항

### 성능

- 샘플 데이터 기준 주요 화면 로딩 2초 이내
- AI 생성 응답 10초 내외 목표

### 보안

- API Key는 서버 환경변수로만 관리
- public GitHub에 비밀키 노출 금지
- 데모 데이터는 가명 또는 샘플 데이터만 사용

### 신뢰성

- AI 응답 실패 시 fallback 문구 제공
- 데이터 파싱 실패 시 명확한 에러 안내

### 사용성

- 비전문가도 3클릭 내 주요 기능 접근 가능
- 상태 배지와 우선순위가 시각적으로 명확해야 함

## 7. 리스크 및 대응

### 리스크

- AI 응답 품질이 들쭉날쭉할 수 있음
- 강의자료 파싱 품질이 낮을 수 있음
- 범위를 넓히면 완성도가 떨어질 수 있음

### 대응

- 회복학습/퀴즈/메시지 생성 포맷을 고정
- 샘플 강의자료를 직접 통제된 형태로 준비
- 규칙기반 핵심 로직 유지
- **기능 추가보다 polish 우선**

## 8. 제출물 체크리스트

### 필수 제출물

- public GitHub repo
- 배포 URL
- AI 리포트 PDF
- 개인정보 수집/이용 동의서 및 참가 각서 PDF

### 코드/배포 체크

- API Key 노출 없음
- .env.example 정리
- README 포함
- 샘플 데이터 포함 여부 점검
- 제출 이후 commit 방지
